Angular History

Angular is a popular open-source web application framework developed and maintained by Google. Its history begins in 2009 when developers Misko Hevery and Adam Abrons created **AngularJS** (originally just Angular), which was designed to simplify the process of building dynamic single-page web applications (SPAs). AngularJS introduced revolutionary concepts like two-way data binding, dependency injection, and directives, which made the development of interactive user interfaces much easier compared to traditional methods. In 2016, Google released a complete rewrite of AngularJS, known as **Angular 2**, which marked a major shift in the framework’s architecture. Unlike AngularJS, which was built using JavaScript, Angular 2 and subsequent versions are built with **TypeScript**, a statically typed superset of JavaScript. This rewrite was done to improve performance, scalability, and maintainability, and it abandoned many of the core ideas of AngularJS in favor of a more modular and component-based architecture. Angular 2 also adopted modern web standards and embraced mobile-first development, leading to better overall performance. After Angular 2, subsequent versions followed a predictable release cycle. Angular 4 was released in 2017 (skipping version 3 to avoid confusion with internal versioning), with improvements to speed and a reduction in the bundle size. Angular continues to evolve with regular updates, adding features like better tooling, Angular Ivy (a new rendering engine), and enhanced developer experience, making it one of the leading frameworks for building modern web applications. Today, Angular is widely used by developers and companies globally, known for its powerful tools, strong community, and long-term support from Google. The framework is now in continuous development, with the goal of providing a robust platform for scalable and maintainable web applications.

Structure of Angular

Angular applications follow a component-based architecture, organized into modular parts. Here are the main building blocks:

  • Modules: NgModules organize an application into cohesive blocks of functionality, such as feature modules or shared modules. Every application has at least one root module.
  • Components: Components are the primary building blocks of the UI in Angular. A component consists of an HTML template, CSS styles, and a TypeScript class that defines the behavior.
  • Templates: Templates are used to define the UI layout and structure. They bind component data to the view and use Angular directives like *ngIf and *ngFor for dynamic content.
  • Services: Services provide specific functionality across components, such as data fetching, logging, and business logic. They are injected into components through Angular's Dependency Injection (DI) system.
  • Directives: Directives are special markers on a DOM element (such as an attribute, element name, or class) that tell Angular's HTML compiler to alter the element's behavior. Structural directives like *ngIf change the layout by adding or removing elements.
  • Routing: Angular's RouterModule is used to configure routes, enabling navigation between different views and components, making it possible to create single-page applications (SPAs).
  • Pipes: Pipes are used to transform data in the template. For example, a pipe can format dates, filter data, or modify text directly in the template.

Component

In Angular, a component is the fundamental building block of the user interface. Each component controls a section of the screen called a view. Components are defined by a class (with properties and methods) and an associated template that renders the view.

Component Structure

The typical structure of an Angular component includes:

  • Class (TypeScript): The component class contains the data and logic that defines the behavior of the component. It is decorated with the Component decorator, which provides metadata about the component.
  • Template (HTML): The template defines the view of the component. It binds to the component's data and interacts with the DOM to display the content. Data binding and directives are used to manipulate the DOM dynamically.
  • Styles (CSS): Each component can have its own styles, which are applied to the component's template. These styles are encapsulated, meaning they apply only to the component and do not affect other components.
  • Metadata (Decorators): The Component decorator is used to define the component's selector, template, and styles. It tells Angular how to use the component.

Example of a Component

Here is an example of a basic Angular component:


        import { Component } from 'angular/core';
        
        @Component({
          selector: 'app-example',
          template: "<div>
                       <h1>{{ title }}</h1>
                       <p>{{ description }}</p>
                     </div>",
          styles: [`
            h1 {
              color: blue;
            }
          `]
        })
        export class ExampleComponent {
          title = 'Hello, Angular!';
          description = 'This is an example of an Angular component.';
        }

Files Created When You Generate a Component in Angular

1. TypeScript File (.ts)

This file defines the component's logic and behavior. It includes the class, properties, and methods that control the component. The Component decorator is used to attach metadata like the selector, template URL, and styles. 


      import { Component } from 'angular/core';
      
      @Component({
        selector: 'app-example',
        templateUrl: './example.component.html',
        styleUrls: ['./example.component.css']
      })
      export class ExampleComponent {
        title = 'Hello, Angular!';
        description = 'This is a simple Angular component example.';
      }

2. HTML Template File (.html)

This file defines the component's view, containing the HTML structure that binds to the component's properties and methods. Angular templates use data binding syntax like {{ and directives like *ngIf and *ngFor. 


      <div>
        <h1>{{ title }}</h1>
        <p>{{ description }}</p>
      </div>

3. CSS File (.css)

This file contains the styles for the component. These styles are scoped to the component, meaning they won’t affect other components unless explicitly shared. 


      h1 {
        color: blue;
        font-size: 24px;
      }
      
      p {
        color: gray;
      }

What is a Standalone Component in Angular?

A standalone component is a new type of component in Angular that does not need to be declared in an NgModule. It simplifies component creation by allowing the component to import dependencies directly, such as CommonModule or FormsModule, and to exist without an associated module.

Features of Standalone Components

  • Can be used without being declared in an NgModule.
  • Directly imports necessary Angular dependencies.
  • Simplifies the app structure by reducing the boilerplate code.

Example of a Standalone Component

Here is an example of a simple standalone Angular component:


    import { Component } from '@angular/core';
    import { CommonModule } from '@angular/common';
    
    @Component({
      selector: 'app-standalone',
      standalone: true,
      imports: [CommonModule],
      template: `
        <div>
          <h1>{{ title }}</h1>
          <p>This is a standalone component in Angular.</p>
        </div>
      `
    })
    export class StandaloneComponent {
      title = 'Hello, Standalone Component!';
    }

In the above example, the @Component decorator includes the standalone: true property, making this component standalone. It also directly imports CommonModule to use standard Angular directives.

What is a Template in Angular?

In Angular, a template is a part of the component that defines the user interface (UI) layout and structure. It is essentially the HTML markup that is used to render the component's view. Templates can include standard HTML elements, Angular-specific directives, and data binding expressions.

Key Features of Angular Templates:

  • Data Binding: Angular templates allow you to bind data from the component class to the view using various types of binding:
    • Interpolation: Uses double curly braces ( {{ }} ) to display component properties.
    • Property Binding: Binds the value of a component property to an HTML element property using square brackets ([ ]).
    • Event Binding: Listens for events (like clicks) and calls methods in the component using parentheses (( )).
    • Two-Way Binding: Combines property and event binding using [( )], typically used with forms.
  • Directives: Templates can use Angular directives to manipulate the DOM, including:
    • Structural Directives: Change the structure of the DOM (e.g., *ngIf, *ngFor).
    • Attribute Directives: Change the appearance or behavior of an existing element (e.g., ngClass, ngStyle).
  • Pipes: Templates can use pipes to transform displayed data (e.g., formatting dates, currency, etc.) using the pipe operator (|).
  • Component Hierarchies: Templates can include other components, enabling a hierarchical structure for creating reusable components.

Example of a Template:

TypeScript Component (example.component.ts)


    import { Component } from '@angular/core';
    
    @Component({
      selector: 'app-example',
      templateUrl: './example.component.html',
      styleUrls: ['./example.component.css']
    })
    export class ExampleComponent {
      title = 'Welcome to Angular!';
      description = 'This is a basic example of an Angular template.';
      items = [ 'Item 1', 'Item 2', 'Item 3' ];
    }

HTML Template (example.component.html)


    <div>
      <h1>{{ title }}</h1>
      <p>{{ description }}</p>
    
      <ul>
        <li *ngFor="let item of items">{{ items }}</li>
      </ul>
    </div>

In the above example, the properties title and description from the component class are displayed using {{ }}, and the *ngFor directive iterates over the items array to generate a list item for each element.

What is a Directive in Angular?

A directive is a class in Angular that allows you to extend the behavior of HTML elements. Directives are used to manipulate the DOM, modify elements, or add functionality to components. There are three types of directives in Angular:

  • Components: Directives with a template.
  • Attribute Directives: Change the appearance or behavior of existing elements.
  • Structural Directives: Change the structure of the DOM by adding or removing elements.

Structural Directives

Structural directives are a special type of directive that alters the layout of the DOM by adding or removing elements. They are denoted by asterisks (*) in templates. Examples of structural directives include:

  • *ngIf: Conditionally includes a template based on the truthiness of an expression.
  • *ngFor: Repeats a template for each item in a collection.
  • *ngSwitch: Conditionally displays one of a set of possible templates based on a matching expression.

Example of Structural Directives

Here is a simple example of using *ngIf and *ngFor:

TypeScript Component (example.component.ts)

import { Component } from '@angular/core';

@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css']
})
export class ExampleComponent {
  showItems = true;
  items = [ 'Item 1', 'Item 2', 'Item 3' ];
}
HTML Template (example.component.html)

        <div>
        <button (click)="showItems = !showItems">Toggle Items</button>

        <ul *ngIf="showItems">
            <li *ngFor="let item of items">{{ items }}</li>
        </ul>
        </div>

In this example, clicking the button toggles the visibility of the list. The *ngIf directive conditionally displays the list based on the showItems property, while the *ngFor directive iterates over the items array to generate list items.

Angular Command Tutorial

This section provides basic commands for starting a new Angular project, creating components, and running the project using the Angular CLI (Command Line Interface).

1. Install Angular CLI

Before starting an Angular project, ensure you have the Angular CLI installed. You can install it globally using npm: 


    npm install -g @angular/cli

2. Create a New Angular Project

To create a new Angular project, use the following command:


    ng new my-angular-project

Replace my-angular-project with your desired project name. You will be prompted to choose some configuration options, such as including Angular routing and selecting a stylesheet format.

3. Navigate to the Project Directory

Once the project is created, navigate into the project directory:


    cd my-angular-project

4. Create a New Component

To generate a new component, use the following command:


        ng generate component my-component

You can also use the shorthand command:


        ng g c my-component

Replace my-component with the name of your component.

5. Run the Angular Project

To start the development server and run the project, use the command: 


        ng serve

By default, the application will be available at {{ localHostUrl }}. You can open this URL in your web browser to see your application running.

6. Building the Project for Production

When you're ready to build your project for production, run:


        ng build --prod

This will create an optimized build of your application in the dist/ folder.
